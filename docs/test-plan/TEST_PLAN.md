## 1. บทนำ

### วัตถุประสงค์
เอกสารนี้อธิบายกลยุทธ์, แนวทาง, ทรัพยากร และตารางเวลาการทดสอบแบบ End-to-End (E2E) ของระบบห้องสมุด เพื่อให้มั่นใจว่าเส้นทางผู้ใช้และฟีเจอร์ทั้งหมดได้รับการทดสอบอย่างครอบคลุมก่อนปล่อยไปใช้งานจริง

### ขอบเขต (Scope)

**ในขอบเขต (In Scope):**
- สมัครสมาชิกและการตรวจสอบตัวตน
- ค้นหาและดูรายการหนังสือ
- การยืมและการคืนหนังสือ
- การจัดการข้อมูลสมาชิก (Members Management)
- จัดการคลังหนังสือสำหรับ Admin
- ระบบการแจ้งเตือน (อีเมล/ในแอปพลิเคชัน)
- ดูรายงานหนังสือค้างคืน (Reports & Overdue)
- การจัดการหนังสือค้างคืน
- การตอบสนองของอุปกรณ์ (Responsive)
- การทดสอบประสิทธิภาพ
- การตอบสนองของอุปกรณ์ (Visual & Responsive)
- การทดสอบบนหลายเบราว์เซอร์ (Cross-Browser)

**นอกขอบเขต (Out of Scope):**
- บูรณาการกับเกตเวย์การชำระเงินภายนอก
- การตั้งค่าบริการอีเมล
- ระบบการจองหนังสือล่วงหน้า (Reservation) และการแจ้งเตือน (Notifications)
- การตั้งค่าโครงสร้างพื้นฐาน
- การปรับปรุงฐานข้อมูล

### แนวทางการทดสอบ
- **ประเภท**: E2E Functional Testing
- **Framework**: Playwright + JavaScript
- **ความครอบคลุม**: 68 Test Cases
- **ระดับการทำให้อัตโนมัติ**: ทั้งหมด (ยกเว้นการทดสอบด้วยตัวเองบางส่วน)

## 2. วัตถุประสงค์ของการทดสอบ

### วัตถุประสงค์หลัก
1. ตรวจสอบการทำงานของเส้นทางผู้ใช้ทั้งหมดจากต้นจนจบ
2. ความถูกต้องของข้อมูลตลอดการไหลของการทำงาน
3. พฤติกรรมของระบบในสถานการณ์ปกติและขอบเขต
4. การถูกต้องของการจัดการข้อผิดพลาดและการตอบรับจากผู้ใช้
5. บูรณาการระหว่างบริการหน้าหน้างานและหลังหน้างาน

### วัตถุประสงค์รอง
1. การสร้างเกณฑ์พื้นฐานของประสิทธิภาพ
2. บันทึกพฤติกรรมของระบบเพื่อการทดสอบการถดถอย
3. สร้างความมั่นใจในความน่าเชื่อถือของระบบ
4. สร้างชุดการทดสอบซ้ำ ๆ สำหรับไปป์ไลน์ CI/CD

## 3. ขอบเขตการทดสอบและความครอบคลุม

### ฟีเจอร์ที่ต้องทดสอบ

#### 3.1 โมดูล Authentication (การตรวจสอบตัวตน)
- สมัครสมาชิก (อีเมล, รหัสผ่าน, การตรวจสอบ)
- เข้าสู่ระบบ (ข้อมูลประจำตัวที่ถูก/ผิด)
- รีเซ็ตรหัสผ่าน
- การจัดการเซสชั่น
- ออกจากระบบ
- ควบคุมการเข้าถึงตามบทบาท (Role-based Access Control)

#### 3.2 โมดูล Search & Discovery (ค้นหาและค้นพบหนังสือ)
- ค้นหาด้วยชื่อ, ผู้แต่ง, ISBN, หมวดหมู่
- กรองตามสถานะความพร้อม
- จัดเรียงตามความเกี่ยวข้อง, วันที่เพิ่ม, ความนิยม
- การแบ่งหน้าผลลัพธ์
- ความถูกต้องของการแสดงรายละเอียดหนังสือ

#### 3.3 โมดูล Borrowing (การยืมหนังสือ)
- ตรวจสอบความพร้อมของหนังสือ
- เริ่มต้นคำขอการยืม
- ยืนยันธุรกรรมการยืม
- การกำหนดวันที่ต้องคืน
- การบังคับใช้ขีดจำกัดการยืม
- ค่าปรับสำหรับหนังสือค้างคืน

#### 3.4 โมดูล Return (การคืนหนังสือ)
- ดูหนังสือที่ยืมไว้ใน Dashboard
- เริ่มต้นการคืน
- ยืนยันธุรกรรมการคืน
- การแจ้งเตือนหนังสือค้างคืน
- การยืนยันและการรับทำบัญชี

#### 3.7 โมดูล Admin (ฟีเจอร์ Admin)
- เพิ่มหนังสือใหม่ไปยังคลัง
- แก้ไขข้อมูลหนังสือ
- ลบหนังสือออกจากคลัง
- ดูการยืมและการคืนทั้งหมด
- สร้างรายงานการจราจร
- จัดการบัญชีผู้ใช้
- ดูการแจ้งเตือนหนังสือค้างคืน

#### 3.9 โมดูล Performance (ประสิทธิภาพ)
- เวลาโหลดหน้า (< 3 วินาที)
- เวลาตอบสนองการค้นหา (< 2 วินาที)
- สามารถรับผู้ใช้พร้อมกันได้ (10-50 คน)
- ประสิทธิภาพของ Query ฐานข้อมูล

## 4. กลยุทธ์การทดสอบ

### 4.1 ระดับการทดสอบ
1. **Unit Testing**: ฟังก์ชัน Code (ความรับผิดชอบของนักพัฒนา)
2. **Integration Testing**: ปฏิสัมพันธ์ระหว่าง Module (เช่น Borrowing กับ Books)
3. **System Testing**: ฟังก์ชันของระบบทั้งหมด
4. **E2E Testing**: เส้นทางการทำงานของผู้ใช้ที่สมบูรณ์ ← **โฟกัสหลัก**
5. **UAT**: การยอมรับของผู้ใช้ปลายทาง

### 4.2 ประเภทของการทดสอบ
- **Functional Testing**: ตรวจสอบว่าฟีเจอร์ทำงานตามที่ต้องการ
- **Negative Testing**: ข้อมูลที่ไม่ถูกต้องและสถานการณ์ข้อผิดพลาด
- **Regression Testing**: ตรวจสอบว่าการแก้ไขไม่ทำให้ฟีเจอร์เดิมเสีย
- **Smoke Testing**: การตรวจสอบเส้นทางสำคัญอย่างรวดเร็ว
- **Performance Testing**: เวลาตอบสนองและการรับผู้ใช้

### 4.3 กลยุทธ์ข้อมูลการทดสอบ
- **ข้อมูลที่เหมือนจริง**: 100+ หนังสือในคลังเพื่อการทดสอบการค้นหา
- **ผู้ใช้ทดสอบ**: หลายบัญชีผู้ใช้ที่มีบทบาทต่างกัน
- **ข้อมูลขอบเขต**: ขีดจำกัดการยืมสูงสุด, นิยามค่าปรับ
- **ข้อมูลที่ไม่ถูกต้อง**: อักขระพิเศษ, SQL Injection, XSS

## 10. การทดสอบและรายงาน

### เมตริกที่ต้องติดตาม
- **Pass Rate**: % Test ที่ผ่าน
- **Coverage**: % ข้อกำหนดที่ครอบคลุม
- **Defect Density**: ข้อบกพร่องต่อ Test Case
- **Execution Time**: ชั่วโมงในการเรียกใช้ชุดทดสอบทั้งหมด
- **Trend**: ผลลัพธ์ทดสอบในหลายการรัน

### เนื้อหารายงานผล
1. บทสรุปสำหรับผู้บริหาร
2. การวิเคราะห์ความครอบคลุม
3. ผลลัพธ์การทดสอบ
4. ข้อบกพร่องและสถานะ
5. การประเมินความเสี่ยง
6. คำสั่งเสนอแนะ
7. ภาคผนวก (ผลลัพธ์โดยละเอียด)
